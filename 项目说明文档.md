# 后台管理系统 & WMS入库管理模块 说明文档

## 项目简介

本系统是一个采用前后端分离架构的后台管理系统，包含基础的用户管理、菜单标签联动功能，并扩展了WMS入库管理模块。系统支持PC端和移动端双端适配，提供响应式界面和优化的移动端交互体验。

---

## 技术栈

### 前端
- **Vue 3**：构建页面组件和响应式交互
- **Element Plus**：提供表单、菜单、标签页、表格等UI组件
- **Vue Router**：路由管理和页面导航
- **Axios**：处理HTTP请求
- **Vite**：前端构建工具和开发服务器
- **@zxing/library**：摄像头扫码解析二维码
- **QRCode**：生成看板二维码

### 后端
- **Spring Boot**：提供RESTful API接口
- **Spring JDBC**：数据库操作
- **MySQL**：数据持久化存储

## 系统功能模块

### 一、基础后台管理系统（作业1）

#### 1. 登录页

**功能说明**：
- 用户身份验证
- 用户名/密码错误提示
- 登录成功后跳转工作台

**界面特性**：
- 左侧系统名称："后台管理系统"
- 右侧登录表单：用户名输入框、密码输入框、登录按钮
- 用户名输入框前使用用户图标
- 密码输入框前使用锁图标，右侧提供显示/隐藏密码的眼睛图标
- 登录按钮使用蓝色主按钮样式
- 页面背景使用浅色渐变，登录卡片居中显示

#### 2. 后台主界面

**布局结构**：
- 左侧菜单栏：动态渲染后端返回的菜单数据
- 顶部标题栏：显示当前页面名称、登录用户、退出登录按钮
- 顶部标签页：多标签页管理，支持切换和关闭
- 中间内容区：展示页面内容

**菜单联动机制**：
- 菜单数据来自后端 `/api/menus` 接口
- 点击菜单项自动打开对应标签页
- 标签页保留访问历史，支持快速切换
- 标签页可关闭（至少保留一个）

**菜单结构**：

| 菜单名称 | 路径 | 组件 | 说明 |
|---------|------|------|------|
| 工作台 | /dashboard | Dashboard | 一级菜单，可直接访问 |
| 系统管理 | 无 | 无 | 一级分组菜单 |
| - 用户管理 | /system/user | UserManage | 系统管理下的二级菜单 |
| - 角色管理 | /system/role | RoleManage | 系统管理下的二级菜单 |
| 标签页示例 | /tabs | TabsDemo | 一级菜单，可直接访问 |

#### 3. 前后端接口对接

| 接口 | 请求方式 | 前端用途 |
|------|---------|---------|
| /api/login | POST | 提交用户名和密码，完成登录 |
| /api/menus | GET | 登录后获取菜单数据 |
| /api/logout | POST | 退出登录并清理会话 |

### 二、WMS入库管理模块（作业2）

#### 1. 模块概述

WMS入库管理模块在后台管理系统基础上扩展，主要功能包括：
- **基础信息管理**：供应商管理和零件管理
- **入库单管理**：创建、编辑、删除入库单，生成并打印入库看板
- **扫码入库**：摄像头扫描看板二维码或手动输入看板号完成入库
- **库存追溯**：查看库存状态和完整的出入库流转记录

#### 2. WMS菜单结构

| 菜单名称 | 路径 | 组件 | 说明 |
|---------|------|------|------|
| 供应商管理 | /wms/supplier | SupplierManage | 增删改查供应商信息 |
| 零件管理 | /wms/part | PartManage | 增删改查零件信息，关联供应商 |
| 入库单列表 | /wms/inbound-order/list | InboundOrderList | 查看、编辑、删除入库单 |
| 创建入库单 | /wms/inbound-order/create | InboundOrderForm | 创建新的入库单 |
| 扫码入库 | /wms/scan | ScanInbound | 扫码看板二维码确认入库 |
| 库存追溯 | /wms/trace | InventoryTrace | 查看库存记录和追溯信息 |

#### 3. 核心功能展示

##### 3.1 供应商管理
- 表格展示供应商列表（编码、名称、联系人、电话等）
- 顶部搜索框和"新增供应商"按钮
- 每行提供编辑和删除操作
- 新增/编辑使用对话框表单

##### 3.2 零件管理
- 零件信息维护（编码、名称、规格、单位、所属供应商）
- 零件关联供应商，下拉选择支持搜索过滤

##### 3.3 创建入库单
- **第一步**：选择供应商、选择仓库
- **第二步**：动态添加入库零件明细
  - 选择零件、输入数量
  - 支持添加多行明细
  - 可随时删除不需要的行
- 提交后后端生成唯一入库单号，返回看板信息供打印

##### 3.4 入库单列表
- 展示所有入库单（单号、供应商、仓库、状态、创建时间）
- **操作按钮**：
  - 修改：仅未完成入库单可修改
  - 删除：仅未完成入库单可删除（需确认）
  - 打印看板：生成并打印看板（含二维码）

**打印看板包含**：
- 入库单号
- 供应商名称
- 零件明细表格
- 二维码（扫码入库使用）

##### 3.5 扫码入库
- **摄像头扫码方式**：
  - 点击"开始扫描"调用摄像头
  - 将看板二维码对准摄像头
  - 自动解析二维码中的看板号
  - 验证后显示入库信息
- **手动输入方式**：
  - 输入看板号进行验证
  - 选择目标库位
  - 确认入库

##### 3.6 库存追溯
- **标签页1 - 追溯记录**：
  - 展示所有出入库操作记录
  - 包含看板号、零件信息、数量、操作类型、操作人、时间
  - 支持按看板号、零件编码、操作类型筛选
- **标签页2 - 当前库存**：
  - 展示各零件当前库存数量
  - 包含零件编码、名称、数量、库位、最后更新时间

### 三、移动端适配优化（新增）

#### 1. 移动端布局策略

**响应式设计**：
- 采用一套代码同时适配PC端和移动端
- 使用媒体查询（Media Queries）实现断点适配
- 三个主要断点：
  - `≤768px`：平板和手机
  - `≤480px`：小屏手机
  - `769px-900px`：平板横屏

#### 2. PC端与移动端交互差异

##### PC端（>768px）
- **左侧**：固定菜单栏（248px宽度）
- **右侧上方**：标题栏 + 退出登录按钮
- **右侧中部**：标签页导航栏（多标签切换）
- **右侧下方**：页面内容区

##### 移动端（≤768px）
- **顶部**：汉堡菜单 + 页面标题
- **点击汉堡菜单**：左侧滑出抽屉式菜单
- **无标签页**：点击菜单直接跳转页面（无标签保留）
- **页面切换**：带淡入动画（0.2s）
- **操作后**：自动关闭抽屉菜单

#### 4. 移动端访问方式

##### 开发环境配置
```javascript
// vite.config.js
export default defineConfig({
  server: {
    host: '0.0.0.0',  // 允许局域网访问
    port: 5173,
    https: true,  // 启用HTTPS（摄像头功能需要）
  }
})
```

##### 访问地址
- **PC端**：`https://localhost:5174/`
- **移动端**：`https://192.168.x.x:5174/`（需接受安全警告）

##### 摄像头使用说明
- **PC端**：使用localhost可直接访问摄像头
- **移动端**：需要HTTPS环境（使用自签名证书）
- **安全警告处理**：
  - Android Chrome：点击"高级" → "继续访问"
  - iPhone Safari：点击"详细信息" → "访问此网站"

## 项目亮点

1. **响应式设计**：一套代码同时支持PC端和移动端，降低维护成本
2. **原生App体验**：移动端采用抽屉菜单、无标签页、页面切换动画，接近原生App体验
3. **摄像头扫码**：支持摄像头实时扫码入库，提高工作效率
4. **看板打印**：自动生成二维码看板，支持扫码追溯
5. **完整追溯**：记录完整的出入库流转历史，支持库存追溯
6. **前后端分离**：清晰的架构设计，便于扩展和维护

## 开发环境要求

- **Node.js**：16+ 版本

- **Maven**：3.6+ 版本
- **MySQL**：8.0 版本

## 注意事项

1. **移动端摄像头**：需要HTTPS环境，开发环境使用自签名证书
2. **浏览器兼容性**：推荐使用Chrome/Edge浏览器
3. **局域网访问**：确保手机和电脑连接同一WiFi
4. **端口占用**：默认使用5173端口，如被占用会自动切换
